## Expected outcomes

### Outcome A — LLM-proposed kernel beats reference end-to-end
*Probability:* low-medium

What we'd see:
```
l3_kernel_o0007  rmsnorm_v2 (LLM-novel)  → 920 tok/s on 1×A100
                  vs reference rmsnorm    → 867 tok/s on 1×A100
                  same regime, dtype, hardware
```

What it answers: Q1 strongly affirmed. The third thesis claim has its
first quantitative empirical support. Even a +3% margin is enough —
it's the architecture's validation, not the magnitude.

### Outcome B — All L3 LLM kernels match or under-perform reference
*Probability:* medium-high

What we'd see: every L3 trial lands within 0.95-1.02× the reference
kernel's serving throughput on the same configuration. No clear
LLM-novel winner.

What it answers: Q1 is honestly negative-or-flat at this surface.
The kernel-into-vLLM architecture works (we now have ~12 datapoints
proving it), but the LLM proposer's *search space* (RMSNorm and
SiluAndMul rewrites) isn't where the wins live. Attention is the
actual hot path. Next session goes to T-21 (attention-layer
injector). The thesis claim isn't refuted — it's narrowed to "the
proposer needs richer search surfaces" rather than "the technique
doesn't work."

### Outcome C — L1 surrogate hit rate ≥30%
*Probability:* medium-high

What we'd see: L1 surrogate kept-rate jumps from the 2026-04-25 run's
0/8 to e.g. 4/10 or 5/10. Possibly an L1 surrogate trial produces a
new global L1 best.

What it answers: Q2 affirmed. ConstrainedOptunaSurrogate validates
the architectural choice. If a surrogate trial also makes the
frontier, the operator-only-finds-wins hypothesis from prior runs
is overturned.

### Outcome D — Reserve-on-stale produces a 2nd-pass improvement
*Probability:* low (needs Q2 to work first, then needs the new region
to genuinely have wins not yet found in 1st-pass)

What we'd see:
```
L2 finds A100×2 win → 14 L1 entries marked stale + reserve granted →
  L1 reserve trial (now with feasibility filter) lands a new
  Pareto entry that's better than the first-pass L1 best
```

What it answers: Q3 affirmed. P4's strongest form has its first
empirical instance.

### Outcome E — Integration breakage
*Probability:* low (smoke validation done) but possible

Examples: Sonnet 4 starts returning unparseable kernel source under
load; FeasibilityModel resampling deadlocks on a configuration we
didn't unit-test; vLLM v1 changes a public interface in a way the
injector's monkeypatch breaks against.

What it answers: a different question — "what's the next bug?". The
per-trial `_vllm.{out,err}` files + events.jsonl + the new fallback
event will surface it concretely. We fix and re-run; cost ~$5 per
debug cycle.
